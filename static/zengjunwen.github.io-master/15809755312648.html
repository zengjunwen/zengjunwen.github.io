设计模式
「简单介绍下」
$[timeformat('2020-02-06T15:52:11+08:00')]

很多时候，我们总听到设计模式这个词，我们都说他重要，为啥重要？我觉得学习设计模式是方便自己和别的开发人员进行有效沟通的重要工具。有可能你已经知道了解决某个问题的方法，并且该方法符合某种设计模式，此时，能够给别人描述解决问题的方法是拥有共同的语言，设计模式就是这种语言。

设计模式（Design pattern）代表了最佳的实践，通常被有经验的面向对象的软件开发人员所采用。设计模式参考书《设计模式 - 可复用的面向对象软件元素》中所提到的设计模式，总共有 23 种。这些模式可以分为三大类：创建型模式（Creational Patterns）、结构型模式（Structural Patterns）、行为型模式（Behavioral Patterns）。我们将通过结合JAVA代码对这些设计模式进行讲解。
