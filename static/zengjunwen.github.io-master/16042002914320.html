Redis数据结构和使用场景

help

help @<name>

String

  • 单值缓存
  • 对象缓存
  • 分布式锁
  • 计数器
  • Web集群session共享
  • 生成分布式系统全局序列号

hash

  • 对象缓存 与string对比 优点：
    1）同类数据归类整合储存，方便数据管理
    2）相比string操作消耗内存与cpu更小
    3）相比string储存更节省空间
    缺点：
    过期功能不能使用在field上，只能用在key上
    Redis集群架构下不适合大规模使用

list

是一个双端链表，可以用作stack、Queue和Blocking MQ（与Queue相比，在没元素时会阻塞，直到队列有消息，可设置超时时间）。

  • 微博和微信公众号消息流

set

  • 微信抽奖小程序
  • 点赞、收藏、标签

zset

有序的集合，跳表实现

  • 排行榜
