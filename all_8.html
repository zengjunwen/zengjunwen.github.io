1. If we have a database with TV episodes, what a good way to model the data if we want to query episodes by year?

Ceate nodes for years and setup relationships between the episode and the year nodes.
We could choose to create a property in the episodes with date. When you want make a query that shows all episodes that were aired in December 2014. If the date was in a property, getting the desired result would involve inspecting and parsing the property, and create some logic to determine the right episodes.

2. What is a good way to come up with a Cypher pattern?

You should picture a pattern in your mind, then translate to ASCII.

Cypher is declarative, that means you specify what you want to retrieve, not how Neo4j should retrieve it.
Easy on the brain, it's not as steep as a learning curve you would have if you started with SQL from scratch. Because use patterns to match the data in the database.
